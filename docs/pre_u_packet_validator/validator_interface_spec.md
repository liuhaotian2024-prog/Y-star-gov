# Validator Interface Spec

## Purpose

Define the future Y-star-gov interface for validating labs-generated Pre-U
Counterfactual Packets before risk-tiered actions proceed.

This interface lets Y-star-gov judge packet validity without generating the
packet or becoming Aiden's subjective brain.

## Inputs

- `packet`: Pre-U Counterfactual Packet object generated by labs.
- `agent_registry_ref`: registry or identity reference used to validate `agent_id`.
- `risk_tier`: action risk tier supplied by hook/future runtime.
- `action_envelope_ref`: requested action envelope or action metadata.
- `hook_context`: optional hook context for boundary/risk decisions.
- `cieu_pre_action_event_ref`: optional pre-action CIEU event reference.

## Outputs

- `validation_status`: one of `valid`, `invalid`, `requires_revision`,
  `requires_escalation`, `warning_only`.
- `failure_codes`: deterministic and semantic failure identifiers.
- `warnings`: non-blocking validation warnings.
- `required_revisions`: concrete packet changes needed before validation.
- `escalation_reason`: reason when operator/human/governance escalation is required.
- `normalized_packet_ref`: stable reference to the packet under validation.
- `cieu_pre_action_requirements`: CIEU linking requirements for hook/future runtime.
- `hook_decision_hint`: one of `allow`, `deny`, `warn`, `escalate`,
  `require_revision`.

## Deterministic Validation Categories

- Schema validity.
- `packet_id` presence.
- `agent_id` validity.
- `agent_capsule_ref` presence.
- `task_id` presence.
- Y* presence.
- `m_functor` presence.
- `x_t_summary` presence.
- `candidate_actions` non-empty.
- `selected_action` references an existing candidate.
- `predicted_r_t1` present for each candidate.
- Residual minimization rationale present.
- Risk/high-risk review fields present.
- `packet_status` valid.
- CIEU linkage policy present.
- Governance expectations present.

## Future Semantic Validation Categories

- `m_functor` grounding via Y* Field validator.
- Y* to mission grounding.
- Action scope check via boundary enforcer.
- Goodhart guard / fake alignment detection.
- Role authorization check.
- Risk-tier escalation check.
- Omission/obligation check when packet absence matters.
- No claim of actual Yt+1 before action.
- No direct DB/log/runtime mutation hidden inside packet.
- No unbounded free-form fantasy replacing structured prediction.

## Required Validator Result Shape

```json
{
  "validation_status": "valid",
  "failure_codes": [],
  "warnings": [],
  "required_revisions": [],
  "escalation_reason": null,
  "normalized_packet_ref": "packet://...",
  "cieu_pre_action_requirements": {
    "should_emit_pre_action_event": true,
    "packet_id_required": true,
    "validation_result_id_required": true
  },
  "hook_decision_hint": "allow"
}
```

The example above is illustrative shape only, not runtime behavior.

## What Y-star-gov Owns

- Deterministic packet validation interface.
- Governance law and boundary validation.
- CZL/CIEU residual semantics.
- Field and m_functor validation boundaries.
- Failure-action taxonomy for hook consumption.

## What Y-star-gov Must Not Own

- Packet generation.
- Aiden subjective cognition.
- Candidate-action imagination.
- Brain DB contents.
- Brain writeback/dream decisions.
- Hook execution itself.
